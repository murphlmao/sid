## ¬ßD Config-file manager (track C)

Systems tab gains a "config files" area: pinned + curated files, click to open
in an in-app editor modal, save when writable. OWNED: `ui/systems_tab.rs`,
`crates/sid-store` (entity + table + facade), `crates/sid/src/ui/mod.rs` if a
new module is cleaner (e.g. `ui/config_editor.rs`).

1. Store: new entity `PinnedFile { path: String }` (identity = path), global
   redb table `pinned_files`, codec V1, list/pin/unpin on `GlobalStore` + the
   `Store` facade (global-only ‚Äî machine-local by nature; workspace-scoped pins
   are future work, note it). Follow the QuickAction table as the pattern.
   Tests mirror the existing entity round-trip tests.
2. Curated candidates (existence-filtered at render/refresh time, shown under a
   "common" header distinct from pins): /etc/fstab, /etc/hosts, /etc/environment,
   /etc/pacman.conf, /etc/ssh/sshd_config, /etc/ssh/ssh_config, /etc/sudoers,
   ~/.ssh/config, ~/.gitconfig, ~/.zshrc, ~/.bashrc, ~/.profile,
   ~/.config/hypr/hyprland.conf, ~/.config/kitty/kitty.conf,
   ~/.config/waybar/config.jsonc. Each row: filename + muted full path +
   pin/unpin affordance; pinned section on top. A small "pin a file‚Ä¶" input
   accepts an arbitrary path (tilde-expanded; nonexistent path ‚áí inline error).
3. Editor: click a row ‚áí modal (surface, large ‚Äî most of the viewport) with the
   file in a multi-line editor ‚Äî reuse the SQL editor's gpui-component Input
   machinery from `ui/db_tab.rs`. Load caps: ‚â§ 1 MiB and valid UTF-8, else a
   read-only notice (mirror `session.rs`'s preview gates). Writability: probe
   with `OpenOptions::new().write(true).open(path)` at open time; read-only ‚áí
   amber banner "read-only ‚Äî needs root; edit with sudoedit" and the save
   affordance disabled. Save: write to a sibling temp file, preserve the
   original's permissions (fs::metadata ‚Üí set_permissions), rename over the
   original; surface errors inline. Esc closes (refocus root_focus!); dirty
   state marks the title; closing dirty asks nothing in v1 (note it).
4. NO sudo/pkexec escalation in v1. No emojis. Theme tokens native
   (`theme::active(cx)`), spacey per ¬ßB.2.
5. Tests: entity/facade round-trip; pure helpers (tilde expansion, curated
   filtering with a fake exists-fn, size/utf8 gate, perms-preserving save via
   tempfile incl. a readonly-file case).

Branch: `round-e/config-files`.
