# Database Slice — Implementation Plan (smart-harvest of dbflux)

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development /
> executing-plans. Steps use `- [ ]`. **Runs concurrently with the SSH slice** (Plan 3C).
> Wave 1 (store + adapter) is disjoint from the SSH track by crate. Wave 2 (UI) follows the
> foundations and the GPUI-ecosystem research.

**Goal:** A daily-usable Database tab — connect to Postgres/SQLite (and browse sid's own
redb store read-only), run SQL, see results in a grid, manage connections in either scope —
built by **harvesting dbflux smartly** rather than from scratch, while keeping sid's own
spine (layered store, keyring secrets, adapter traits).

**Architecture (smart-harvest, decided 2026-07-01):** Drivers come from the POC's clean
`sid-db-clients` adapter (already fits sid's trait/store/secret shape), NOT dbflux's
SQLite-fused driver layer. The DB UI adopts **`gpui-component`** (crates.io, gpui-0.2.2 —
gives text-input, tree-sitter SQL highlighting, tables, tree, theme) and lifts dbflux's two
near-drop-in generic widgets (`data_table`, `document_tree`, MIT-attributed) where
`gpui-component` doesn't already cover it. Everything is wired against sid's `Store`
connection facade + `sid-db` adapter + `sid-secrets` keyring — dbflux's own domain model
(one SQLite DB, no layered scope) is **not** imported; its feature panels are reference-only.

**Tech Stack:** Rust 2024 · gpui 0.2.2 · gpui-component 0.5 · sid-core (`DbKind` + `DbClient`
trait) · new `sid-db` crate (tokio-postgres 0.7, rusqlite 0.40 bundled) · sid-store · sid-secrets.

## Global Constraints
- **sid's spine is non-negotiable.** Connections persist through the **attributive layered
  store** (global redb + per-workspace committed `.sid/config.toml`); secrets live in the OS
  keyring referenced by opaque `secret_ref` (never in DSN/config). Adapter pattern: GPUI only
  in `crates/sid`; DB drivers only in `sid-db`; the `DbClient` trait seam lives in `sid-core`.
- **`DbKind` already exists** at `sid_core::db::DbKind { Postgres, Sqlite, Redb }` (label
  round-trip) — the shared contract. Do not redefine it.
- **postcard is positional** — schema changes to stored entities go through the codec version
  byte with an explicit migration (mirror the just-merged `Host` v2 work exactly).
- **Pragmatic TDD (standing rule):** TDD the load-bearing logic (facade CRUD/promote/demote,
  connection migration, descriptor DSN round-trip, sqlite open/query, error mapping) with one
  good check per behavior. No exhaustive permutations. UI is observation-gated, never
  unit-tested. Keep the suite fast.
- **License:** any file copied from dbflux (MIT OR Apache-2.0, © 2025 Ignacio Perez) keeps an
  attribution header + a `THIRD_PARTY_NOTICES` entry. `gpui-component` is a normal dependency.
- Commits: conventional, no Co-Authored-By trailer.

## WAVE 1 — Foundations (parallel now; disjoint crates)

### Track DS — Store connection facade  *(crate: `sid-store`; sonnet)*

#### DS1 — Enrich `DbConnection` + versioned migration
- **Files:** `crates/sid-store/src/entities.rs`, `crates/sid-store/src/global.rs`, `Cargo.toml`
  (add `sid-core = { path = "../sid-core" }`); tests in `tests/roundtrip.rs`, `tests/global.rs`.
- **New:** add to `DbConnection`: `#[serde(default)] pub kind: DbKind` (from `sid_core::db`)
  and `#[serde(default)] pub name: String` (display label; `id` stays the identity key). Add a
  retained `DbConnectionV1` legacy shape + `CONNECTION_VERSION = 2` + a `decode_connection`
  version branch (v1 → `From<DbConnectionV1>` defaulting `kind: Postgres`, `name: id.clone()`),
  and switch `list_connections`/`get_connection`/`upsert_connection` to the versioned path —
  **exactly mirroring commit 6c13622's Host v2 migration**. Other entities untouched.
- **Tests:** v1 connection payload decodes with `kind == Postgres`, `name == id`; v2 round-trip
  for each `DbKind`; TOML round-trip; a v1 store reopens and lists cleanly.

#### DS2 — `WorkspaceStore::remove_connection`
- **Files:** `crates/sid-store/src/workspace.rs`; tests in `tests/workspace.rs`.
- **New:** `pub fn remove_connection(&self, id: &str) -> Result<bool>` mirroring `remove_host`
  (load → retain-by-id → save; returns whether present). (Workspace TOML `[[db.connection]]`
  and `upsert_connection` already exist.)
- **Tests:** remove present → true + gone after reload; remove absent → false.

#### DS3 — `Store` connection facade
- **Files:** `crates/sid-store/src/store.rs`; tests in `tests/store.rs`.
- **New (each a structural copy of the corresponding host method):**
  `read_connections(&self, scope: &Scope, filters: ViewFilters) -> Result<Vec<Attributed<DbConnection>>>`,
  `write_connection(&self, c: &DbConnection, scope: &Scope) -> Result<()>`,
  `delete_connection(&self, id: &str, scope: &Scope) -> Result<bool>`,
  `promote_connection(&self, id: &str, from: &WorkspaceId) -> Result<()>`,
  `demote_connection(&self, id: &str, to: &WorkspaceId) -> Result<()>` (promote/demote keep the
  same `StoreError::Conflict` guard as hosts). The composer is already generic — reuse `compose`.
- **Tests:** mirror the host suite — write-lands-in-named-layer-only; read composes + dedups;
  delete-from-workspace-leaves-global-and-unshadows; promote/demote move + conflict-guard.
- **Deliverable:** the store's connection write side is symmetric with hosts, behind the facade.

### Track DA — DB adapter crate  *(crates: `sid-core`, new `sid-db`; sonnet)*

#### DA1 — Port the `DbClient` trait + result types into `sid-core`
- **Files:** `crates/sid-core/src/db.rs` (extend — `DbKind` already here); `sid-core/Cargo.toml`
  (async-trait already present).
- **Port near-verbatim** from POC `db_client.rs`: `DbError`, `OpenParams`, `SqliteMode`,
  `Column`/`ColumnType`, `Row`, `PageCursor`, `QueryPage`, `SchemaInfo`/`TableInfo`, the
  `DbClient` async trait (`open`/`close`/`execute`/`query_paged`/`schema_introspect`/`cancel`/
  `kind`) and `DbClientDescriptor` (`kind`/`connection_fields`/`assemble_params`/
  `dsn_to_field_values`) + `ConnField`/`ConnFieldKind`. Drop the POC's `DbKind::ConfigReader`
  variant naming in favor of `sid_core::db::DbKind::Redb`.
- **Tests:** POC-style doctests + trait object-safety compile checks (`Arc<dyn DbClient>`).

#### DA2 — `sid-db`: Postgres + SQLite clients (port)
- **Files:** new crate `crates/sid-db/{Cargo.toml,src/lib.rs,src/postgres.rs,src/sqlite.rs,
  src/descriptor.rs,src/lexer.rs}`; workspace `Cargo.toml` members + `tokio-postgres`/`rusqlite`
  deps (carry the POC's dep comments).
- **Port near-verbatim** POC `sid-db-clients/src/{postgres,sqlite,descriptor,lexer}.rs`
  (imports move to `sid_core::db`). Keep the load-bearing bits: Postgres spawned
  connection-driver task + `CancelToken` + `Drop`-abort; the `query_paged` subquery wrap (keep
  its known single-SELECT limitation + comment); SQLite `spawn_blocking` + `OpenExisting`
  no-create safety; the descriptors' password-never-in-DSN routing.
- **Tests (pragmatic):** descriptor DSN round-trip + garbage-input non-panic; sqlite
  open-existing/create + a real `query_paged` against a temp file; `map_*_error` tables. Postgres
  network paths are manual/`#[ignore]`d.

#### DA3 — `Redb` read-only browse engine
- **Files:** `crates/sid-db/src/redb_browse.rs`.
- **New:** a `DbClient` impl over an opened `sid_store` (read-only) exposing tables as
  "schema" and rows via `query_paged` (no SQL parsing — list tables / dump a table). `execute`
  returns `DbError::Invalid` (read-only). This is the POC `ConfigReader` idea, natively.
  Keep it minimal — `// ponytail: table dump only; no query language over redb`.
- **Tests:** open a temp redb store with seeded rows → `schema_introspect` lists tables →
  `query_paged` returns rows.
- **Deliverable:** `sid-db` provides `DbClient` for all three min engines behind `sid-core`.

## WAVE 2 — Database tab UI  *(SKETCH — detail after Wave 1 + the GPUI-ecosystem research lands)*

Deferred to a detail pass because its tasks depend on (a) the merged facade/adapter APIs and
(b) the awesome-gpui / `gpui-component` capability findings (which widgets are adopt-vs-lift).
Firm shape:

- **DU-TLS — Postgres TLS *(SECURITY — blocks shipping the DB tab against remote hosts)*.**
  `sid-db`'s Postgres client currently connects with `NoTls` (ported from the POC) — cleartext
  credentials + data on the wire, flagged HIGH by security review. Before the connection flow
  can target a remote host: add a rustls (or native-tls) connector, honor the DSN's `sslmode`
  (`disable` → NoTls only; `require`/`verify-full` → TLS with platform trust-store cert
  validation), and restrict `NoTls` to `localhost`/`127.0.0.1`/unix-socket. Not live-exploitable
  until the tab wires a connect, but a hard prerequisite for daily-use quality. (Marked at the
  code site in `crates/sid-db/src/postgres.rs`.)
- **DU0 — Adopt `gpui-component`** as a `crates/sid` dependency; spike its text editor +
  tree-sitter SQL highlighter + table + tree against sid's palette (observation-gated).
  Research-validated (2026-07-01): dbflux AND pgui both ship on the same crates.io
  `gpui 0.2.x` + `gpui-component 0.5.x` pairing sid pins — strong precedent this library carries
  the whole DB UI surface. No GPUI project factors its DB widgets into a reusable crate, so
  `gpui-component` is the dependency + dbflux is the only license-clean (MIT/Apache) source to
  *lift* the 2 generic widgets from. Pattern-only references (do NOT copy — license/coupling):
  pgui (no license), zqlz (GPL-3 + git-gpui), based/zedis (git-gpui).
- **DU1 — Connection registry + form** (descriptor-driven, from DA1's `ConnField`s), writing
  through `Store::write_connection` with `save to: workspace|global` (reuse the host form's
  save-to + secret-lifecycle pattern; password → keyring, `secret_ref` on the record).
- **DU2 — Left tree sidebar** (schema browser — connections → databases → tables), lifting
  dbflux's `document_tree` generic if `gpui-component`'s tree is insufficient. **This supersedes
  the mockup's "connection dropdown, not a left tree" rule — update the design-doc note.**
- **DU3 — SQL editor** on `gpui-component`'s highlighter; **DU4 — results grid** (virtualized;
  `gpui-component` table or lifted dbflux `data_table`); **DU5 — run/paging/export/history**
  wiring `execute`/`query_paged` on a background task → results, mirroring the POC's
  spawn→outcome→apply data flow (reference `wire/database.rs`, do not copy its ratatui).
- **DU6 — gate:** observation checklist (connect PG/SQLite/redb, run query, page, edit/save
  connection in each scope, secret in keyring not config), adversarial review of the connect/
  secret path, THIRD_PARTY_NOTICES for any lifted dbflux code.
