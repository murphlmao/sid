**Goal:** Second-generation interaction system for sid — side-pane forms with framed fields,

## Approved design decisions (binding for all branches)

1. **Form surface = side pane.** Add/edit forms and detail/inspector views render in a right
   side pane (~40/60 list/pane split) inside the current tab. No more centered popup modals
   for forms. The old modal substrate stays only for tiny confirms (delete? discard?).
2. **Forms look like forms.** Framed input boxes per field, label gutter, validation errors
   in red under the offending box, segmented Choice buttons. `Info` sections (read-only
   key→value rows, muted) are visually distinct from `Editable` sections — never let a user
   wonder whether a row is a setting or a fact.
3. **Dynamic reshaping.** A form spec carries a reshape hook; changing a watched field (e.g.
   Kind: Postgres→SQLite) rebuilds the section list. Values of surviving keys are preserved.
4. **Context-aware Tab (the "smart switcher").**
   - List focus: `Tab` / `Shift+Tab` (and `Ctrl+Tab` where the terminal reports it) cycle the
     top tab strip. `→` enters the right pane. `Enter` opens/edits.
   - Pane focus: `Tab` / `Shift+Tab` move fields, `←` (on non-text fields) or `Esc` leaves
     back to the list — with a discard-confirm if the form is dirty. `Enter` saves.
   - Footer hints re-render per focus state; the current meaning of Tab is always on screen.
5. **Kitty keyboard protocol** enabled at startup via crossterm keyboard-enhancement flags so
   `Ctrl+Tab` / `Ctrl+Enter` work in kitty/wezterm/ghostty/foot. Universal fallbacks: `Shift+Tab`
   (prev tab), `O` (background-open).
6. **`+ add new` row** at the top of every list panel (Database connections, Workspaces, SSH
   hosts, Quick Actions). Accent-styled, cursor-selectable, Enter opens the add form. Governed
   by behavior toggle `show_add_new_row` (default on). `N` keeps working regardless.
7. **Background tabs:** `Ctrl+Enter` (or `O`) opens a detail tab without switching to it —
   `TabManager::push_background`.
8. **Workspaces model:** workspace = single repo or umbrella (a real git repo, e.g. gen4-stack)
   + registered satellites. Overview keeps inline tree expansion (`→` expand, `↓` walk in,
   `←` collapse); `Enter` on workspace or satellite opens it as a pushed tab (subrepo view —
   same layout for umbrella and single repo). Detail tab: umbrella git header (branch · dirty ·
   outgoing), left list (umbrella row + satellites, real GitProvider data, off-thread), right
   drill-in stack (ops → Outgoing commits / Log / Branches / Stash / Worktrees → commit list →
   scrollable diff; `←` pops). Registration: create-new wizard with feature checklist; adopt-existing
   directory scan (finds nested + symlinked repos, pre-checked multi-select confirm).
9. **Network detail:** side pane with Info section (MAC, IPs, link state, RX/TX, default-route
   badge) + Editable section limited to sid-level prefs (pin-to-top, display alias). OS-level
   interface config explicitly out of scope; the pane says so.
10. **Database form:** Name + Kind (segmented); Postgres → Host/Port/Database/User/Password,
    SQLite → Path picker. `Ctrl+T` test-connection through `DbClient` off-thread. Edit-in-place
    via Enter on an existing connection. Generated DSN shown as read-only Info row.
11. **Testing policy for this effort (overrides CLAUDE.md per-commit rigor, per Murphy):**
    test the code each task adds — targeted `cargo test -p <crate> <module>` — not the full
    pipeline. One comprehensive gate review (`/sid-gate --full` + clippy + fmt across the
    workspace) happens once, at the very end, as its own task in this master plan.
12. **No `Co-Authored-By: Claude` trailers on commits.**
13. **Help overlay (`?`).** A summonable keybind cheatsheet — global chords plus the active
    tab's bindings — opened with `?` from list focus (`Ctrl+?` also accepted on kitty-protocol
    terminals). Footers slim down: only the 3-4 contextual verbs plus `?: help`; everything
    else lives in the overlay. Built once in the substrate (branch 0); branches keep their
    footer_hint lists short and rely on the overlay for the long tail.
