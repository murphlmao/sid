## §A Secrets: keyring-or-nothing, prompt per login (track S)

Murphy's decision: the passphrase-on-every-launch vault UX is dead. New model:
**keyring → memory**. If there's no keyring, sid does not persist credentials; it asks
for the password at connect time ("password per login").

1. `sid-secrets/src/resolve.rs`
   - `SecretBackendToggles` loses `file_enabled`; chain: keyring enabled + probe pass
     → `Keyring`, else → `Memory` + warning + recommendation (keep the
     `gnome-keyring` install hint; drop the "enable encrypted-file" wording).
   - Remove `BackendKind::EncryptedFile`. Keep `file.rs` **in the crate untouched**
     (crypto reviewed & tested; document at its module head: "dormant — not wired
     into the chain since 2026-07-03; passphrase-per-launch UX rejected").
   - Update the resolve test matrix accordingly.
2. `sid-store` `Settings.secret_file_enabled`: DO NOT remove the field (postcard v3
   positional encoding). Doc it as ignored/dormant.
3. `crates/sid/src/app.rs`
   - `open_secrets`: no vault path, no file handle in the return.
   - Delete the unlock-modal plumbing: `secret_unlock` field, subscription,
     `open_secret_unlock` / `close_secret_unlock` / `on_secret_unlock_event`, the
     render call, and `crates/sid/src/ui/secret_unlock.rs` + its `mod` decl.
   - **Banner → badge.** Remove the persistent "secrets: …" status line wherever it
     renders. Replace with a small badge at the right end of the primary tab strip,
     shown ONLY when degraded (memory fallback or keyring disabled): a rounded
     `pill` with `!` (WARN yellow 0xd0c88a). Click toggles a small anchored popover
     (same deferred/anchored pattern as other overlays) showing the full debug text
     from `secret_status_message` (backend, warning, recommendation). Healthy keyring
     ⇒ nothing rendered at all.
4. **Connect-time password prompt.** New `crates/sid/src/ui/password_prompt.rs`:
   minimal modal — title "password for {label}", one masked `TextInput`, Enter =
   submit, Esc = cancel. No emoji. Hook points:
   - SSH: where `ssh_connect::resolve_secret` fails (dangling ref + Password auth)
     or `connect_params` would fail with "no secret was resolved": instead of
     surfacing the error, open the prompt; on submit pass the password directly
     into the connect attempt (one shot, never written to config), and *if* a
     `secret_ref` exists also `secrets.put(ref, bytes)` so the session remembers.
   - DB: same treatment where `resolve_db_secret` yields a dangling ref for a
     connection whose kind needs auth (postgres). Retry the action that triggered.
   - Keep secrets out of logs; hold plaintext only as long as the connect needs.
5. Copy updates: host/db forms' password helper text when the backend is memory:
   "no OS keyring — passwords last this session only; you'll be asked at connect".
6. Tests: updated resolve matrix; a pure `needs_password_prompt(...)` decision fn
   with unit tests; keep `secret_status_message` tests (it now feeds the popover).

## §B Network: filter hotkey + typed sortable columns (track N)

1. `keymap.rs`: `Action::FocusFilter` (label "Find / Filter"), bindings
   `Ctrl+F` (NormalOnly) and `Ctrl+/` (`chord("/", true, None)`, NormalOnly).
   Add to `ALL_ACTIONS`. `app.rs` dispatch: when `active_tab == Tab::Network`,
   focus the network tab's shared filter input (add a `focus_filter(window, cx)`
   on `NetworkTabState`); other tabs: no-op for now.
2. Sortable columns on the four tables (Ports, Services, Docker, Kubernetes) using
   gpui-component's native support: `Column::sortable()` +
   `TableDelegate::perform_sort` (check the 0.5.1 signature in
   `~/.cargo/registry/src/*/gpui-component-0.5.1/src/table/`). Typed comparators —
   never lexicographic on numbers:
   - Ports: port (numeric), proto, addr, process (case-insensitive), pid (numeric)
   - Services: name (ci), state (active-first rank: active < activating < reloading
     < inactive < failed, then alpha), sub-state, description
   - Docker: name, image, state (running < paused < exited), status, ports
   - Kube pods: namespace, name, ready, phase (Running first), restarts (numeric), node
3. Sort state lives in each delegate; `recompute()` applies filter *then* the active
   sort so filtering never clears the sort. Clicking a header toggles asc/desc.
4. Tests: pure comparator tests per delegate (e.g. ports 9 < 10 < 443 < 8080;
   services active-first; docker running-first). Sort+filter composition test.

## §C Systems tab v1 (track Y)

`Tab::System` currently renders a placeholder/stand-in. Build v1: local system
overview + processes. Read-only except kill.

1. `sid-core/src/sys.rs`: add `SystemOverview` (hostname, kernel, os, uptime_secs,
   load_avg (f64,f64,f64), cpu_total_pct, cpu_per_core: Vec<f32>, mem_total/used,
   swap_total/used — bytes) and a required `fn overview(&mut self) ->
   Result<SystemOverview, SysError>` on `SysProvider` (update the doc example).
   For processes reuse `list_processes`/`ProcessInfo`; if `ProcessInfo` lacks
   cpu_pct/mem_bytes/user, extend it (fix all constructors) — do NOT invent a
   parallel type.
2. `sid-sysinfo`: implement via the existing `sysinfo` dep (already has system,
   disk, user features). Two refresh calls needed for meaningful cpu% — keep the
   cached `System` between calls per the trait contract.
3. `crates/sid/src/ui/systems_tab.rs`: top overview cards (CPU total + per-core
   bars, memory + swap bars, load/uptime/host line) + a processes Table
   (gpui-component) with sortable typed columns (cpu desc default, mem, pid, name,
   user) and the shared filter-input pattern from network_tab. Kill action wires
   through the existing `kill_process` path — reuse the pid-0 / i32-overflow guards
   verbatim (do not duplicate; call the same helper the ports table uses).
   Refresh every 2s only while the tab is active (follow network_tab's async
   refresh pattern). Galaxy-dark styling consistent with other tabs; no emoji.
4. `app.rs`: render the tab; keep `Action::Settings`'s stand-in note honest.
5. Tests: pure formatters (bytes humanize, uptime), sysinfo mapping, comparator
   tests. UI gated by observation, per CLAUDE.md.
