# UX-v2 Branch 6: Real SecretStore — OS Keyring Backend + Migration

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development or superpowers:executing-plans. Steps use checkbox syntax for tracking.

**Goal:** Replace the plaintext-in-redb `PlainStore` with an OS keyring-backed `KeyringStore` impl of the existing `SecretStore` trait. Add a one-time migration that moves every secret out of the redb `secrets` table into the keyring on first enable, deleting the redb copies after verified write. Wire a `USE_OS_KEYRING` settings key into startup so `main.rs` selects the right backend; fall back to `PlainStore` with a warning toast if the keyring daemon is unavailable.

**Architecture:** `KeyringStore` is a new struct in `crates/sid-secrets/src/keyring_store.rs`. A `migrate_to_keyring` free function in `crates/sid-secrets/src/migration.rs` handles the one-time move. Both types live exclusively in `sid-secrets` — the `keyring` crate name never appears in any other crate. `crates/sid/src/wire.rs` gets a `build_secret_store` helper that reads `USE_OS_KEYRING` from the store and returns `Arc<dyn SecretStore>`; `main.rs` calls this helper instead of the current inline construction. Tests mock the keyring via a `FakeKeyring` seam defined in the crate's `#[cfg(test)]` module; no real OS keyring call ever fires in CI.

**Tech Stack:** `keyring = "3"` (crates.io). Chosen because: it is the canonical Rust crate for OS keyring access, targets libsecret (Linux secret-service D-Bus API) and Keychain (macOS), carries an MIT licence (allowed by `deny.toml`), and the v3 API (`Entry::new(service, user)` → `set_password` / `get_password` / `delete_credential`) is stable and well-maintained. Alternative `secret-service` targets Linux only. Windows is explicitly out of scope (spec §"Keyring integration"). The `keyring` dep lives only in `crates/sid-secrets/Cargo.toml` per the adapter pattern.

//! OS keyring-backed [`SecretStore`] implementation.
//!
//! `KeyringStore` maps every [`SecretId`] to a `keyring::Entry` under the
//! service name `"sid"`.  On Linux this talks to the secret-service D-Bus
//! API (libsecret / gnome-keyring / kwallet); on macOS it uses the Keychain
//! API.  Windows is not supported.
//!
//! # Testability seam
//!
//! `KeyringStore` is generic over `B: KeyringBackend`, a trait that mirrors
//! the three `keyring::Entry` methods we use.  Production code instantiates
//! `KeyringStore::new()` which uses the real `keyring` crate; tests inject
//! `FakeKeyring` (defined in the `#[cfg(test)]` support module).
//!
//! # Examples
//!
//! ```no_run
//! use sid_secrets::keyring_store::KeyringStore;
//! use sid_core::adapters::secrets::{SecretId, SecretStore};
//!
//! let store = KeyringStore::new();
//! let id = SecretId::new("db.local-pg.password");
//! store.put(&id, b"s3cr3t").unwrap();
//! assert_eq!(store.get(&id).unwrap().unwrap(), b"s3cr3t".to_vec());
//! ```

use sid_core::adapters::secrets::{SecretError, SecretId, SecretStore};

/// Abstraction over the three `keyring::Entry` operations used by
/// [`KeyringStore`].  The trait exists solely to allow a `FakeKeyring`
/// substitute in tests so CI never hits the real OS keyring daemon.
///
/// Production code uses `OsKeyringBackend`; tests use `FakeKeyring`.
pub trait KeyringBackend: Send + Sync {
    /// Store `secret` bytes under `key`.  The implementation is responsible
    /// for encoding bytes to the string form the OS keyring requires.
    fn set(&self, key: &str, secret: &[u8]) -> Result<(), String>;

    /// Retrieve the bytes stored at `key`, or `None` if absent.
    fn get(&self, key: &str) -> Result<Option<Vec<u8>>, String>;

    /// Remove the entry at `key`.  Idempotent — absent key is `Ok(())`.
    fn delete(&self, key: &str) -> Result<(), String>;

    /// Return all keys currently held in this store.
    ///
    /// For the OS backend this is unsupported (keyring APIs don't expose
    /// enumeration); the impl returns `Err("list_ids unsupported by OS
    /// keyring backend")`.  `FakeKeyring` returns the live key set.
    fn list_keys(&self) -> Result<Vec<String>, String>;
}

/// Production keyring backend: delegates to [`keyring::Entry`].
///
/// Bytes are hex-encoded before storage because the OS keyring stores
/// UTF-8 strings, not raw byte arrays.
pub struct OsKeyringBackend;

const SERVICE: &str = "sid";

impl KeyringBackend for OsKeyringBackend {
    fn set(&self, key: &str, secret: &[u8]) -> Result<(), String> {
        let hex = hex_encode(secret);
        keyring::Entry::new(SERVICE, key)
            .map_err(|e| format!("entry create: {e}"))?
            .set_password(&hex)
            .map_err(|e| format!("set_password: {e}"))
    }

    fn get(&self, key: &str) -> Result<Option<Vec<u8>>, String> {
        let entry = keyring::Entry::new(SERVICE, key)
            .map_err(|e| format!("entry create: {e}"))?;
        match entry.get_password() {
            Ok(hex) => {
                let bytes = hex_decode(&hex)
                    .map_err(|e| format!("hex decode for '{key}': {e}"))?;
                Ok(Some(bytes))
            }
            Err(keyring::Error::NoEntry) => Ok(None),
            Err(e) => Err(format!("get_password: {e}")),
        }
    }

    fn delete(&self, key: &str) -> Result<(), String> {
        let entry = keyring::Entry::new(SERVICE, key)
            .map_err(|e| format!("entry create: {e}"))?;
        match entry.delete_credential() {
            Ok(()) => Ok(()),
            Err(keyring::Error::NoEntry) => Ok(()),
            Err(e) => Err(format!("delete_credential: {e}")),
        }
    }

    fn list_keys(&self) -> Result<Vec<String>, String> {
        Err("list_ids is not supported by the OS keyring backend; \
             use migration helpers which track ids separately"
            .into())
    }
}

fn hex_encode(bytes: &[u8]) -> String {
    bytes.iter().map(|b| format!("{b:02x}")).collect()
}

fn hex_decode(s: &str) -> Result<Vec<u8>, String> {
    if s.len() % 2 != 0 {
        return Err(format!("odd hex length {}", s.len()));
    }
    (0..s.len())
        .step_by(2)
        .map(|i| {
            u8::from_str_radix(&s[i..i + 2], 16)
                .map_err(|e| format!("invalid hex at {i}: {e}"))
        })
        .collect()
}

/// OS keyring-backed [`SecretStore`].
///
/// Generic over `B: KeyringBackend` so tests can inject a [`FakeKeyring`]
/// without touching the real keyring daemon.  Use [`KeyringStore::new`] for
/// production; [`KeyringStore::with_backend`] for tests.
///
/// # Note on `list_ids`
///
/// The OS keyring does not expose enumeration.  `KeyringStore::list_ids`
/// returns `Err(SecretError::Storage(...))` with a clear message.  Callers
/// that need a list (e.g., the migration helper) must obtain ids from the
/// source store (`PlainStore::list_ids`) rather than asking the keyring.
///
/// # Examples
///
/// ```no_run
/// use sid_secrets::keyring_store::KeyringStore;
/// use sid_core::adapters::secrets::{SecretId, SecretStore};
///
/// let store = KeyringStore::new();
/// let id = SecretId::new("ssh.passphrase.ed25519");
/// store.put(&id, b"pass").unwrap();
/// assert!(store.get(&id).unwrap().is_some());
/// store.delete(&id).unwrap();
/// assert!(store.get(&id).unwrap().is_none());
/// ```
pub struct KeyringStore<B = OsKeyringBackend> {
    backend: B,
}

impl KeyringStore<OsKeyringBackend> {
    /// Construct a `KeyringStore` backed by the real OS keyring.
    ///
    /// # Examples
    ///
    /// ```no_run
    /// use sid_secrets::keyring_store::KeyringStore;
    /// let _store = KeyringStore::new();
    /// ```
    pub fn new() -> Self {
        Self {
            backend: OsKeyringBackend,
        }
    }
}

impl Default for KeyringStore<OsKeyringBackend> {
    fn default() -> Self {
        Self::new()
    }
}

impl<B: KeyringBackend> KeyringStore<B> {
    /// Construct with an injected backend (for tests).
    ///
    /// # Examples
    ///
    /// ```
    /// use std::collections::HashMap;
    /// use std::sync::Mutex;
    /// use sid_secrets::keyring_store::{KeyringBackend, KeyringStore};
    /// use sid_core::adapters::secrets::{SecretId, SecretStore};
    ///
    /// struct NullBackend;
    /// impl KeyringBackend for NullBackend {
    ///     fn set(&self, _k: &str, _v: &[u8]) -> Result<(), String> { Ok(()) }
    ///     fn get(&self, _k: &str) -> Result<Option<Vec<u8>>, String> { Ok(None) }
    ///     fn delete(&self, _k: &str) -> Result<(), String> { Ok(()) }
    ///     fn list_keys(&self) -> Result<Vec<String>, String> { Ok(vec![]) }
    /// }
    ///
    /// let store = KeyringStore::with_backend(NullBackend);
    /// assert!(store.get(&SecretId::new("a")).unwrap().is_none());
    /// ```
    pub fn with_backend(backend: B) -> Self {
        Self { backend }
    }
}

impl<B: KeyringBackend> SecretStore for KeyringStore<B> {
    fn put(&self, id: &SecretId, value: &[u8]) -> Result<(), SecretError> {
        self.backend
            .set(id.as_str(), value)
            .map_err(SecretError::Storage)
    }

    fn get(&self, id: &SecretId) -> Result<Option<Vec<u8>>, SecretError> {
        self.backend
            .get(id.as_str())
            .map_err(SecretError::Storage)
    }

    fn delete(&self, id: &SecretId) -> Result<(), SecretError> {
        self.backend
            .delete(id.as_str())
            .map_err(SecretError::Storage)
    }

    fn list_ids(&self) -> Result<Vec<SecretId>, SecretError> {
        self.backend
            .list_keys()
            .map(|ks| ks.into_iter().map(SecretId::new).collect())
            .map_err(SecretError::Storage)
    }
}

//! One-time migration: move secrets from [`PlainStore`] (redb) to a
//! [`KeyringStore`] (OS keyring).
//!
//! # Semantics
//!
//! - For each id returned by `source.list_ids()`:
//!   1. Read the value from `source`.
//!   2. Write it to `dest`.
//!   3. Read back from `dest` and assert the bytes match (verified write).
//!   4. On verified success, delete from `source`.
//! - On partial failure: keep unmigrated entries in `source` (don't delete
//!   them).  The migration is idempotent — a subsequent call re-attempts
//!   every id still present in `source`; already-migrated ids have already
//!   been deleted so they are not double-written.
//! - If a secret is present in both backends after a previous partial
//!   migration (power loss between step 2 and step 4), the keyring copy
//!   wins — the write in step 2 is idempotent (overwrite), and the verify
//!   step will succeed, so the source entry is deleted cleanly on re-run.
//!
//! # Examples
//!
//! ```
//! use std::sync::Arc;
//! use sid_secrets::PlainStore;
//! use sid_secrets::keyring_store::KeyringStore;
//! use sid_secrets::migration::{MigrateResult, migrate_to_keyring};
//! use sid_core::adapters::secrets::{SecretId, SecretStore};
//! use sid_store::{OpenStore, RedbStore, Store};
//! use tempfile::tempdir;
//!
//! let dir = tempdir().unwrap();
//! let inner: Arc<dyn Store> =
//!     Arc::new(RedbStore::open(&dir.path().join("sid.redb")).unwrap());
//! let source = PlainStore::new(Arc::clone(&inner));
//! let id = SecretId::new("db.pw");
//! source.put(&id, b"password").unwrap();
//!
//! # #[derive(Default)]
//! # struct FakeKeyring(std::sync::Mutex<std::collections::HashMap<String, Vec<u8>>>);
//! # impl sid_secrets::keyring_store::KeyringBackend for FakeKeyring {
//! #     fn set(&self, k: &str, v: &[u8]) -> Result<(), String> {
//! #         self.0.lock().unwrap().insert(k.into(), v.into()); Ok(()) }
//! #     fn get(&self, k: &str) -> Result<Option<Vec<u8>>, String> {
//! #         Ok(self.0.lock().unwrap().get(k).cloned()) }
//! #     fn delete(&self, k: &str) -> Result<(), String> {
//! #         self.0.lock().unwrap().remove(k); Ok(()) }
//! #     fn list_keys(&self) -> Result<Vec<String>, String> {
//! #         Ok(self.0.lock().unwrap().keys().cloned().collect()) }
//! # }
//! let dest = KeyringStore::with_backend(FakeKeyring::default());
//! let result = migrate_to_keyring(&source, &dest).unwrap();
//! assert_eq!(result.migrated, 1);
//! assert_eq!(result.failed, 0);
//! // Source entry removed after verified write.
//! assert!(source.get(&id).unwrap().is_none());
//! ```

use sid_core::adapters::secrets::{SecretError, SecretId, SecretStore};
use tracing::{info, warn};

/// Outcome of a `migrate_to_keyring` call.
///
/// `migrated + failed` equals the number of ids that existed in the source
/// at the start of the call.
///
/// # Examples
///
/// ```
/// use sid_secrets::migration::MigrateResult;
///
/// let r = MigrateResult { migrated: 3, failed: 1, errors: vec!["x".into()] };
/// assert_eq!(r.migrated + r.failed, 4);
/// ```
#[derive(Debug, Default)]
pub struct MigrateResult {
    /// Number of secrets successfully moved to the keyring and removed from
    /// the source store.
    pub migrated: usize,
    /// Number of secrets that could not be migrated (write, read-back, or
    /// delete failed).  These remain in the source store.
    pub failed: usize,
    /// Human-readable error messages, one per failure.
    pub errors: Vec<String>,
}

/// Move all secrets from `source` to `dest`.  Returns a [`MigrateResult`]
/// describing how many entries were migrated and how many failed.
///
/// On partial failure (`result.failed > 0`) the caller should surface
/// `result.errors` to the user (toast or log) and can safely retry later —
/// this function is idempotent.
///
/// # Errors
///
/// Returns `Err(SecretError)` only if `source.list_ids()` itself fails.
/// Per-secret failures are accumulated in [`MigrateResult::errors`].
///
/// # Examples
///
/// ```
/// use std::sync::Arc;
/// use sid_secrets::PlainStore;
/// use sid_secrets::keyring_store::KeyringStore;
/// use sid_secrets::migration::migrate_to_keyring;
/// use sid_core::adapters::secrets::{SecretId, SecretStore};
/// use sid_store::{OpenStore, RedbStore, Store};
/// use tempfile::tempdir;
///
/// let dir = tempdir().unwrap();
/// let inner: Arc<dyn Store> =
///     Arc::new(RedbStore::open(&dir.path().join("sid.redb")).unwrap());
/// let source = PlainStore::new(inner);
///
/// # #[derive(Default)]
/// # struct FakeKeyring(std::sync::Mutex<std::collections::HashMap<String, Vec<u8>>>);
/// # impl sid_secrets::keyring_store::KeyringBackend for FakeKeyring {
/// #     fn set(&self, k: &str, v: &[u8]) -> Result<(), String> {
/// #         self.0.lock().unwrap().insert(k.into(), v.into()); Ok(()) }
/// #     fn get(&self, k: &str) -> Result<Option<Vec<u8>>, String> {
/// #         Ok(self.0.lock().unwrap().get(k).cloned()) }
/// #     fn delete(&self, k: &str) -> Result<(), String> {
/// #         self.0.lock().unwrap().remove(k); Ok(()) }
/// #     fn list_keys(&self) -> Result<Vec<String>, String> {
/// #         Ok(self.0.lock().unwrap().keys().cloned().collect()) }
/// # }
/// let dest = KeyringStore::with_backend(FakeKeyring::default());
/// // Empty source — migration is a no-op.
/// let r = migrate_to_keyring(&source, &dest).unwrap();
/// assert_eq!(r.migrated, 0);
/// assert_eq!(r.failed, 0);
/// ```
pub fn migrate_to_keyring(
    source: &dyn SecretStore,
    dest: &dyn SecretStore,
) -> Result<MigrateResult, SecretError> {
    let ids = source.list_ids()?;
    let total = ids.len();
    info!(total, "starting secret migration to keyring");

    let mut result = MigrateResult::default();

    for id in ids {
        match migrate_one(source, dest, &id) {
            Ok(()) => {
                result.migrated += 1;
            }
            Err(e) => {
                let msg = format!("failed to migrate '{}': {}", id.as_str(), e);
                warn!("{msg}");
                result.errors.push(msg);
                result.failed += 1;
            }
        }
    }

    info!(
        migrated = result.migrated,
        failed = result.failed,
        "secret migration complete"
    );
    Ok(result)
}

fn migrate_one(
    source: &dyn SecretStore,
    dest: &dyn SecretStore,
    id: &SecretId,
) -> Result<(), String> {
    // Step 1: read from source.
    let value = source
        .get(id)
        .map_err(|e| format!("source read: {e}"))?
        .ok_or_else(|| format!("source entry disappeared mid-migration"))?;

    // Step 2: write to dest.
    dest.put(id, &value)
        .map_err(|e| format!("dest write: {e}"))?;

    // Step 3: verified read-back.
    let read_back = dest
        .get(id)
        .map_err(|e| format!("dest read-back: {e}"))?
        .ok_or_else(|| format!("dest read-back returned None immediately after write"))?;

    if read_back != value {
        return Err(format!(
            "read-back mismatch: wrote {} bytes, read back {} bytes",
            value.len(),
            read_back.len()
        ));
    }

    // Step 4: delete from source.
    source
        .delete(id)
        .map_err(|e| format!("source delete: {e}"))?;

    Ok(())
}

/// Build the active [`SecretStore`] implementation.
///
/// Reads [`sid_store::settings_keys::USE_OS_KEYRING`] from `store`.  When
/// `true`, constructs a [`sid_secrets::KeyringStore`] and probes the keyring
/// daemon with a round-trip write/read/delete of a sentinel key.  On probe
/// failure, falls back to [`sid_secrets::PlainStore`] and logs a warning.
///
/// Returns `(store, used_keyring)` — the bool lets `main.rs` push a
/// fallback toast when the keyring was requested but unavailable.
///
/// # Examples
///
/// ```
/// use std::sync::Arc;
/// use sid::wire::build_secret_store;
/// use sid_store::{OpenStore, RedbStore, Store, TypedSettings};
/// use tempfile::tempdir;
///
/// let dir = tempdir().unwrap();
/// let redb = Arc::new(RedbStore::open(&dir.path().join("s.redb")).unwrap());
/// // Default: USE_OS_KEYRING absent → PlainStore returned, used_keyring false.
/// let (secrets, used_keyring) = build_secret_store(
///     &*redb,
///     Arc::clone(&redb) as Arc<dyn Store>,
/// );
/// assert!(!used_keyring);
/// ```
pub fn build_secret_store(
    store: &RedbStore,
    plain_arc: Arc<dyn Store>,
) -> (Arc<dyn sid_core::adapters::secrets::SecretStore>, bool) {
    use sid_store::TypedSettings;

    let want_keyring = store
        .get_bool(sid_store::settings_keys::USE_OS_KEYRING)
        .unwrap_or(None)
        .unwrap_or(false);

    if want_keyring {
        let ks = sid_secrets::KeyringStore::new();
        // Probe: write + read-back + delete a sentinel.
        let probe_key = sid_core::adapters::secrets::SecretId::new("sid.__keyring_probe");
        let probe_ok = ks
            .put(&probe_key, b"probe")
            .and_then(|_| ks.get(&probe_key))
            .and_then(|v| {
                if v.as_deref() == Some(b"probe".as_slice()) {
                    Ok(())
                } else {
                    Err(sid_core::adapters::secrets::SecretError::Storage(
                        "probe read-back mismatch".into(),
                    ))
                }
            })
            .and_then(|_| ks.delete(&probe_key));

        match probe_ok {
            Ok(()) => {
                tracing::info!("keyring probe succeeded; using KeyringStore");
                return (Arc::new(ks), true);
            }
            Err(e) => {
                tracing::warn!(
                    "keyring probe failed ({}); falling back to PlainStore",
                    e
                );
            }
        }
    }

    let plain = sid_secrets::PlainStore::new(plain_arc);
    (Arc::new(plain), false)
}
