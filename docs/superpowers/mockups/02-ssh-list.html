SSH tab — Key Manager drawer open over the host detail pane.
Answers "how do I manage SSH keys for each connection?" — the [K: keys] action on a host opens this drawer, which lists every key in ~/.ssh/, shows which hosts each is used by, flags weak/legacy keys (rsa-2048), and exposes generate / show / regenerate / delete.
Answers "how do I reset my known_hosts?" — not on this view; lives under [X: debug] (next to [K: keys]), which opens a debug pane with "prune known_hosts entry for selected host" and "reset known_hosts" actions, so a stale fingerprint is one keystroke away from being cleared.
Read-only ssh-config hosts are marked (cfg) and dimmed so it's obvious which entries sid owns vs which come from ~/.ssh/config.
