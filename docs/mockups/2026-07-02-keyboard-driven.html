✦ sid — keyboard-driven

One modifier story, everywhere. Plain accelerators outside a terminal; the same binding plus Shift once a terminal is focused (kitty convention), so the muscle memory transfers instead of fighting the terminal for its own bindings. This document is the decided keymap — render it, argue with it, then it goes in Settings → Keymap.

aPrinciples

Four rules the rest of this document follows without exception.

1
One modifier storyPlain accelerators outside the terminal. Inside a focused terminal, the same binding + Shift — kitty's convention — so muscle memory transfers instead of resetting per-surface.
2
Tab is for forms, onlyTab/Shift+Tab is reserved for field cycling inside forms. Zone/pane cycling is F6/Shift+F6 — never overload Tab with two jobs.
3
Everything is in the paletteEvery action, connection, and tab is reachable via Ctrl+K. The keymap is a set of shortcuts to the palette, not a separate surface.
4
Every binding is yoursRebindable in Settings → Keymap, with live conflict detection and a one-click reset to default, per row or all at once.

bBindings, by context

Grouped by where focus is. The full set also renders as a condensed cheat-sheet — press ? anywhere outside a text field to see it.

⚠ Collision surfaced by principle 1: outside a terminal, Ctrl+Shift+Tab means "previous session tab." Apply the same-binding-+-Shift rule to the terminal's own Ctrl+Tab ("next session tab") and you land on the same chord, Ctrl+Shift+Tab — now meaning "next" inside a terminal. Two actions, one binding. See the highlighted row in the Keymap pane below; this needs Murphy's call (options in the closing section).

?Open questions for Murphy

  1. The Ctrl+Shift+Tab collision (§b/§d). Same-binding-+-Shift is clean everywhere else, but it doubles up here because outside-terminal already uses Shift for "prev." Two ways out: (a) inside a terminal, drop Tab for session-cycling entirely and use kitty's own default, Ctrl+Shift+←/→; or (b) keep Tab outside only, and give inside-terminal cycling a distinct chord that isn't a Shift-of-a-Shift. Leaning (a) since it's already kitty's real default and doesn't invent a new chord — but it's a one-context exception to principle 1, worth saying out loud before it's load-bearing.
  2. Do the primary-tab jumps need a terminal-focused variant? Ctrl+1…5 are plain outside a terminal; inside one, is Ctrl+Shift+1…5 worth reserving to jump tabs without first hitting Ctrl+Shift+A to release focus, or is that one extra keystroke fine as the deliberate "are you sure you're leaving the terminal" speed bump?
  3. ? while another overlay is open. Right now this mockup just lets it stack (cheat-sheet over palette). In the real app, should ? be swallowed while the palette or a rebind-capture is active, since a bare ? might legitimately be part of a search query?
  4. Two-step Delete — does "armed" expire? First press arms, second confirms. Worth deciding whether the armed state auto-disarms after a few seconds of inactivity, or only via Esc/moving selection — affects whether it needs a visible countdown.
