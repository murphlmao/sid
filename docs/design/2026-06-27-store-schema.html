3 · Decisions locked

Layout: git-centric — global items + registry + sessions in redb; each workspace's config in its committed .sid/config.toml. One Store API hides the two read paths.
Composition: attributive union, never override. Default view collapses true dupes (workspace primary); "hide global" is a separate opt-in. Both via checkboxes; storage stays lossless.
Encoding: postcard for redb (fast binary, native fields); TOML for the committed file (git-diffable). Secrets in the OS keyring via secret_ref.
